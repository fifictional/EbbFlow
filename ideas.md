speech_to_text_suggest — When the user is stuck (long pause with no typing, or <200ms burst → long pause pattern), suggest trying Google Docs Voice Typing (Tools → Voice Typing). The article's #1 recommendation.

anti_perfectionism — Different from encourage. Catches the "retyping same sentence 5 times" pattern (high backspace density + slow progress) and shows: "Get the bad draft down. You can edit later." Targets perfectionism/RSD specifically.

outline_prompt — When long pauses + many corrections suggest the user is overwhelmed by too many ideas mid-paragraph, show: "Drop 3 bullet points of what you want to say before writing." Could even open a small text box for the outline.

working_memory_rescue — When a long pause (>5s) follows active typing (user likely forgot their sentence mid-way), show: "Your last word was '___'. Try voice typing to capture it before it's gone." This maps to the article's working memory section.

template_offer — If the user is writing something repetitive (lots of starts/stops at the beginning of a doc), offer: "Save this as a template so next time you skip the blank page." Addresses decision fatigue.
